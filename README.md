# Repro: `nuxt-og-image@6.5.1` `#og-image` alias resolves to non-existent `dist/shared/runtime/`

Minimal Nuxt 4 project showing that a takumi OG component scaffolded with the
official CLI imports `#og-image/app/utils`, which the module's own
`#og-image` alias points at a directory that does not exist in the published
package. Typecheck fails with TS2307; production build fails with
`vite:load-fallback ENOENT`.

## Steps to reproduce

```bash
npm install
npm run typecheck
```

1. `postinstall` runs `nuxi prepare`, generating `.nuxt/tsconfig.app.json`
   with the module's `#og-image` path map.
2. `nuxi typecheck` resolves `app/components/OgImage/Hero.takumi.vue` and
   follows its `import { useOgImageRuntimeConfig } from '#og-image/app/utils'`.
3. The build crashes with:

   ```
   app/components/OgImage/Hero.takumi.vue: error TS2307:
     Cannot find module '#og-image/app/utils' or its corresponding type declarations.
   ```

4. Run `npm run build` to see the matching production-build failure:

   ```
   [vite:load-fallback] Could not load
     /…/node_modules/nuxt-og-image/dist/shared/runtime/app/utils
     (imported by app/components/OgImage/Hero.takumi.vue):
     ENOENT: no such file or directory
   ```

## Expected behaviour

The CLI-emitted import resolves. Typecheck and build succeed.

## Actual behaviour

Both fail. The `#og-image` alias points at a directory that exists in the
package layout (hashed bundle chunks) but does not contain the
`app/utils.{js,d.ts}` subtree the CLI-emitted import expects.

## Root cause

`.nuxt/tsconfig.app.json` (generated by `nuxi prepare`):

```jsonc
"paths": {
  "#og-image": ["../node_modules/nuxt-og-image/dist/shared/runtime"]
}
```

That directory contains only hashed bundle chunks
(`nuxt-og-image.*.mjs`) — no `runtime/app/utils.{js,d.ts}`. The actual
runtime tree is at `node_modules/nuxt-og-image/dist/runtime/`.

The CLI rewrite step that produces the broken import is in
`node_modules/nuxt-og-image/dist/cli.mjs:57`:

```js
} else if (sourceValue === '#og-image/shared') {
  const specifiers = node.specifiers || []
  const hasRuntimeConfig = specifiers.some((s) => {
    const imported = s.imported || s.local
    return imported?.name === 'useOgImageRuntimeConfig'
  })
  if (hasRuntimeConfig) {
    replacements.push({
      start: source.start + 1,
      end: source.end - 1,
      text: '#og-image/app/utils',
    })
  }
}
```

The CLI rewrites `#og-image/shared` → `#og-image/app/utils`, but the
`#og-image` alias is mapped to `dist/shared/runtime/` instead of
`dist/runtime/`. Aligning the alias with `dist/runtime/` (or pointing
`#og-image/app/utils` at `dist/runtime/app/utils` explicitly) resolves it.

No existing issue or PR addresses this misalignment at the time of writing.
