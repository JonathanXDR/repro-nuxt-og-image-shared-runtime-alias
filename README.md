# Repro: `nuxt-og-image@6.5.1` `#og-image` alias resolves to non-existent `dist/shared/runtime/`

Minimal Nuxt 4 project that triggers both a typecheck failure and a production-build failure when a takumi OG component is in the tree.

## What you should see

### `npm run typecheck` (auto-runs on boot)

```
app/components/OgImage/Hero.takumi.vue: error TS2307:
  Cannot find module '#og-image/app/utils' or its corresponding type declarations.
```

### `npm run build`

```
[vite:load-fallback] Could not load
  /…/node_modules/nuxt-og-image/dist/shared/runtime/app/utils
  (imported by app/components/OgImage/Hero.takumi.vue): ENOENT: no such file or directory
```

## Why it fails

`.nuxt/tsconfig.app.json` (generated by the module) declares:

```jsonc
"paths": {
  "#og-image": ["../node_modules/nuxt-og-image/dist/shared/runtime"]
}
```

But that directory contains **only hashed bundle chunks** — no `runtime/app/utils.{js,d.ts}`. The actual runtime tree lives at `dist/runtime/`. The CLI-scaffolded template (`dist/cli.mjs:57`) emits `import … from '#og-image/app/utils'`, hitting both the missing type and the missing on-disk file.

## Versions

- `nuxt@4.4.6`
- `nuxt-og-image@6.5.1`
